# JHAP 랜딩 페이지 프로젝트 스펙

> 운동생리학 학습자를 위한 전환 최적화 랜딩 페이지

---

## 1. 프로젝트 목적

### 핵심 목표
- **단일 시험 → 운동생리학 전체 학습자**로 타겟 확장
- 자격증 취득 + 지식 습득을 동시에 제공하는 학습 브랜드
- 구매 전환율 최적화 (방문 → 관심 → 확신 → 구매)

### 타겟 사용자
- 건강운동관리사, 스포츠지도사 (전문/생활/유소년/노인/장애인)
- 체육/스포츠 전공생
- 운동생리학 자기계발 학습자

---

## 2. 브랜드 아이덴티티

### JHAP의 가치
- 국가공인 체육 자격증 취득 지원
- 운동생리학 핵심 개념 정리
- 몸에 대한 과학적 이해 제공

### 톤앤매너
- 전문적이지만 친근하게
- 과학적이지만 이해하기 쉽게
- 신뢰감 + 접근성

## 3. 디자인 원칙

### 3.1 색상 전략
**목적**: Primary 색상 남용 방지 → 전환 집중

| 색상 | 용도 | 이유 |
|------|------|------|
| **Emerald** (#10b981) | CTA 버튼 + Hero 강조 | 전환 포인트 명확화 |
| **Blue** | 정보/업데이트 섹션 | 신뢰감, 공식성 |
| **Slate** | 통계, 아이콘, 중립 요소 | 가독성, 전문성 |
| **Amber** | 별점 | 직관적 인식 |
| **Emerald/Rose** | 상태 표시 (추천/비추천) | 명확한 구분 |

**핵심 규칙**: Primary는 사용자가 클릭해야 할 곳에만 사용

### 3.2 타이포그래피
**목적**: 가독성 + 시각적 위계

- **제목 (H1-H2)**: `font-bold` + `leading-[1.3]` + `tracking-tight`
- **소제목 (H3-H4)**: `font-semibold` + `tracking-[-0.01em]`
- **본문**: `leading-[1.7]` (편안한 읽기)
- **버튼**: `font-semibold` (명확한 액션)

### 3.3 간격 시스템
**목적**: 일관성 + 리듬감

- **섹션**: `py-12 md:py-16` (Hero 제외)
- **카드**: `p-6` (모든 카드 통일)
- **버튼**: `min-h-[44px]` (터치 접근성)

### 3.4 인터랙션
**목적**: 피드백 + 몰입감

- **카드 호버**: `-translate-y-1` + `shadow-md`
- **애니메이션 딜레이**: 1.3배 증가 (자연스러운 흐름)
- **확장 카드**: 페이지 이동 없이 인라인 표시

## 4. 페이지 구조 및 전략

### 4.1 섹션 순서와 목적

```
1. Hero
   목적: 첫인상, 핵심 가치 전달
   전략: Primary 색상 강조 ("한 권으로")

2. Stats (통계)
   목적: 사회적 증거, 신뢰 구축
   전략: 숫자 + 별점으로 객관성

3. FitCheck (추천/비추천)
   목적: 자기합리화 유도, 환불 방지
   전략: 2열 카드로 즉시 매칭

4. StudyMethod (학습 가이드)
   목적: 구매 후 불안 해소
   전략: 3단계 명확한 프로세스

5. WhyJHAP (차별점)
   목적: 경쟁 우위 강조
   전략: 아이콘 + 간결한 메시지

6. Reviews (후기)
   목적: 사회적 증거 강화
   전략: 실제 사용자 경험

7. WorkbookPreview (자료 미리보기)
   목적: 구체적 정보 제공
   전략: 확장 카드 (페이지 이동 없음)

8. TrustPolicy (신뢰 정책)
   목적: 구매 장벽 제거
   전략: 전달/환불/업데이트 명시

9. Urgency (긴급성)
   목적: 구매 지연 방지
   전략: Blue 색상으로 정보성 강조

10. PurchaseProcess (구매 절차)
    목적: 프로세스 투명화
    전략: 4단계 시각화

11. FAQ
    목적: 마지막 의문 해소
    전략: Accordion으로 간결하게
```

### 4.2 전환 흐름
```
방문 → 관심 (Hero, Stats)
     → 확신 (FitCheck, Reviews, WhyJHAP)
     → 행동 (Urgency, CTA)
     → 완료 (PurchaseProcess)
```

## 5. 주요 의사결정 기록

### 5.1 모달 → 확장 카드
**문제**: Next.js static export에서 동적 라우트 불가  
**결정**: 인라인 확장 카드 방식 채택  
**이유**:
- Static export 호환
- 페이지 이동 없이 빠른 정보 확인
- 모바일 UX 개선

### 5.2 Primary 색상 제한
**문제**: 초록색 남용으로 시선 분산  
**결정**: CTA와 Hero 강조에만 사용  
**이유**:
- 전환 포인트 명확화
- 전문적인 느낌 강화
- 색상 피로도 감소

### 5.3 애니메이션 딜레이 증가
**문제**: 너무 빠른 애니메이션으로 어지러움  
**결정**: 모든 딜레이 1.3배 증가  
**이유**:
- 자연스러운 시선 흐름
- 콘텐츠 인지 시간 확보
- 프리미엄 느낌

### 5.4 아이콘 통일 (Lucide)
**문제**: HTML 엔티티 혼용으로 일관성 부족  
**결정**: 모든 UI 아이콘을 Lucide로 통일  
**이유**:
- 일관된 디자인 언어
- 크기/색상 제어 용이
- 접근성 개선

### 5.5 컴포넌트 분리
**문제**: HomeTab 400+ 줄로 유지보수 어려움  
**결정**: 11개 섹션 컴포넌트로 분리  
**이유**:
- 코드 가독성 향상
- 재사용성 증가
- 팀 협업 용이

## 8. 콘텐츠 원칙

### 8.1 FitCheck (추천/비추천)
- **추천**: 구체적 상황 (자격증 준비, 단기 대비)
- **비추천**: 명확한 제외 (강의 원하는 분, 기초 없는 분)
- **목적**: 구매 후 불만 방지

### 8.2 Urgency (긴급성)
- **메시지**: 최신 기준 반영, 시험 시즌 대비
- **색상**: Blue (경고 아닌 정보)
- **목적**: 구매 지연 방지

### 8.3 Trust Policy
- **포함**: 전달 시간, 재다운로드, 업데이트, 환불
- **형식**: Notice Card
- **목적**: 구매 불안 해소
